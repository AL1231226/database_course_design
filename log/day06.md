## 一、管理员发货反馈

商品发货后管理员可在订单详情弹窗中填写发货反馈（如发货日期、物流信息、预计送达等），用户在自己的订单卡片底部能看到反馈内容，管理员也可随时修改。

**数据库改动：** orders 表新增 `admin_feedback TEXT` 字段，init.sql 同步更新。
**后端：** OrderMapper 加 `updateFeedback` 方法，OrderController 新增 `PUT /api/orders/{id}/feedback` 接口。
**前端：** OrderManage.vue 详情弹窗加反馈文本框和保存按钮，OrderHistory.vue 订单卡片底部展示反馈区域（有反馈才显示）。

## 二、顾客修改密码

登录顾客在导航栏可看到"修改密码"按钮，点击弹出修改密码对话框，输入原密码和新密码即可修改。管理员看不到该按钮。

**实现方式：** 后端 `PUT /api/auth/password` 接口从 JWT 取 userId，校验原密码后用 BCrypt 加密新密码存入数据库。前端 CustomerLayout.vue 导航栏加"修改密码"弹窗，`v-if="user.role === 'customer'"` 控制仅顾客可见。

## 遇到的问题

### 1. 下单时间不对（日期差一天 + 没有时分秒）

**问题：** 订单管理页显示的下单时间不正确。三个原因叠加：

1. **时区问题：** 前端用 `new Date().toISOString().slice(0, 10)` 取日期，`toISOString()` 返回 UTC 时间（中国 UTC+8），北京时间凌晨下单会显示前一天。
2. **数据库类型问题：** `order_date` 字段是 `DATE` 类型，只存日期不存时间，传了时分秒也被截断。
3. **Jackson 序列化格式：** Java `Date` 默认输出 ISO 8601 格式（如 `2026-06-11T00:00:00.000Z`），不是可读格式。

**解决：**

- 前端从 UTC 日期改成本地时间带时分秒的字符串
- 数据库 `ALTER TABLE orders MODIFY order_date DATETIME`，init.sql 同步
- `Orders.java` 的 `orderDate` 加 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")`

### 2. 订单时间入库差 8 小时

**问题：** 提交订单时传 `2026-06-11 14:17:23`（GMT+8），数据库里存成了 `2026-06-11 06:17:23`。

**原因：** 数据库连接配置 `application.yaml` 里 `serverTimezone=UTC`，JDBC 把 Jackson 按 GMT+8 解析好的时间又转成了 UTC 写入。

**解决：** 将 `application.yaml` 的 `serverTimezone=UTC` 改为 `serverTimezone=Asia/Shanghai`。已入库的偏差数据通过 SQL `DATE_ADD(order_date, INTERVAL 8 HOUR)` 修复。
